# 小红书适配器

## 目标

小红书是果汁哥的图文内容平台，以图片+文字笔记为主，适合查询生活分享类和视觉化呈现的知识内容。

## 官方账号

- **账号名**：guozhige
- **平台**：小红书（xiaohongshu.com / RedNote）
- **主页 URL**：`https://www.xiaohongshu.com/user/profile/guozhige`（实际 URL 需以平台 UID 为准）
- **App 搜索**：在小红书 App 内搜索用户名 `guozhige`
- **内容类别**：日常分享

## 查询策略

### 主要查询路径

1. **用户主页笔记列表**：访问 `xiaohongshu.com/user/profile/{user_id}`，获取笔记列表
2. **搜索入口**：`https://www.xiaohongshu.com/search_result?keyword=guozhige&type=user`
3. **App 优先**：小红书 Web 端部分内容需要登录，App 内浏览体验更佳

### 内容类型识别

| 小红书特征 | 对应 content_type |
|------------|------------------|
| 多图+文字 | `image_post` |
| 单图+文字 | `image_post` |
| 视频笔记 | `video` |
| 纯文字笔记 | `short_post` |

## 适合提取的字段

| 字段 | 可用性 | 提取说明 |
|------|--------|---------|
| `publish_time` | ✅ 中 | 笔记列表中显示发布时间，部分以相对时间展示 |
| `title` | ✅ 高 | 笔记标题通常显示在封面图下方 |
| `summary` | ✅ 中 | 笔记正文摘要（Web 端可能截断） |
| `content_type` | ✅ 高 | 图文 / 视频笔记在列表中有区分标志 |
| `source_locator` | ✅ 高 | 格式：`https://www.xiaohongshu.com/explore/{note_id}` |
| `like_count` | ✅ 高 | 点赞数公开显示 |
| `collect_count` | ✅ 高 | 收藏数公开显示 |
| `image_urls` | ✅ 中 | 封面图 URL |

## "今天"查询处理

1. 访问用户主页笔记列表（默认时间降序排列）
2. 遍历笔记，筛选当日发布的内容
3. 小红书时间显示规则：
   - "X分钟前" / "X小时前"：近期发布，换算为绝对时间
   - "昨天"：昨日，具体时间不明，标注为前一天
   - "MM月DD日"：当年对应日期
   - "YYYY年MM月DD日"：历史内容
4. 若笔记列表第一条已非当日，说明今日无更新

## 跨平台去重

- **与微博**：标题或内容高度相似时进行去重
- 小红书笔记较为独立，重复发布率较低
- 若发现同内容，保留小红书版本（因为图片质量通常更高）

## 失败处理

| 失败场景 | 处理方式 |
|----------|----------|
| Web 端访问受限（需登录） | 说明：小红书部分内容需要在 App 内查看。请在小红书 App 内搜索用户 `guozhige` |
| 用户主页 404 | 说明账号 ID 可能已更新，建议通过搜索入口重新定位 |
| 笔记详情无法加载 | 记录封面图和标题，标注"详情页无法加载，内容可能不完整" |
| 图片无法加载 | 不影响文字内容提取，图片字段设为 null |

## 注意事项

- 小红书 Web 端对非登录用户限制较多，完整浏览体验依赖 App
- 账号名 `guozhige` 可能不是唯一的，确认时需检查主页头像和简介
- 小红书笔记的 note_id 可从 URL 中提取：`xiaohongshu.com/explore/{note_id}`
