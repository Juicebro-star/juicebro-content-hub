# 微博适配器

## 目标

微博是果汁哥更新频率最高的平台，以短帖和图文为主，通常是最新动态的第一发布地。本适配器覆盖微博公开内容的查询策略。

## 官方账号

- **账号名**：果汁哥6688
- **平台**：微博（weibo.com）
- **主页 URL**：`https://weibo.com/u/` + 用户 UID（需先确认 UID）
- **搜索入口**：`https://s.weibo.com/user?q=果汁哥6688`
- **内容类别**：日常分享

## 查询策略

### 主要查询路径

1. **用户主页时间线**：访问微博用户主页（`weibo.com/{username}` 或 UID 路径），获取最新发帖列表
2. **微博搜索**：若主页访问受限，通过 `s.weibo.com/user?q=果汁哥6688` 定位账号
3. **API 方式（可选）**：微博有公开 API，但需要注册开发者账号，作为 V2 扩展点

### 内容类型识别

| 微博特征 | 对应 content_type |
|----------|------------------|
| 纯文字（< 280 字） | `short_post` |
| 含图片 | `image_post` |
| 含视频链接 | `video` |
| 含文章链接 | `article`（需跟进链接） |

## 适合提取的字段

| 字段 | 可用性 | 提取说明 |
|------|--------|---------|
| `publish_time` | ✅ 高 | 微博显示绝对时间（历史）或相对时间（近期），如"2小时前"需换算 |
| `title` | ❌ | 短帖无标题，设为 null |
| `summary` | ✅ 高 | 直接提取微博正文，超长截断为 150 字 |
| `content_type` | ✅ 高 | 根据媒体附件类型判断 |
| `source_locator` | ✅ 高 | 格式：`https://weibo.com/{uid}/{mid}` |
| `like_count` | ✅ 高 | 公开可见 |
| `comment_count` | ✅ 高 | 公开可见 |
| `repost_count` | ✅ 高 | 公开可见 |
| `image_urls` | ✅ 中 | 图片链接列表 |
| `video_url` | ✅ 中 | 视频播放链接 |

## "今天"查询处理

1. 访问用户主页时间线
2. 遍历帖子，筛选 `publish_time` 在当日 00:00 - 23:59（北京时间）的内容
3. 微博的相对时间格式：
   - "X分钟前"：当前时间减去分钟数
   - "X小时前"：当前时间减去小时数
   - "昨天 HH:mm"：昨日对应时间
   - "MM-DD HH:mm"：当年对应日期时间
   - "YYYY-MM-DD HH:mm"：绝对时间

## 跨平台去重

- **与快手/抖音**：短视频内容可能同步，以标题+时长为 `dedupe_hash` 基础
- **与小红书**：图文内容可能同步，以标题+前200字摘要为 `dedupe_hash` 基础
- 去重时保留最早发布的平台版本，其他版本填入 `also_posted_on`

## 失败处理

| 失败场景 | 处理方式 |
|----------|----------|
| 主页无法访问（403/空白） | 提示用户直接打开 `weibo.com` 搜索"果汁哥6688" |
| 相对时间无法解析 | 标注 `publish_time: null`，在 `extraction_notes` 中说明 |
| 内容被折叠（需登录展开） | 提取可见部分，在 `extraction_notes` 标注"内容可能不完整" |
| 账号主页内容需登录 | 输出：⚠️ 微博部分内容需要登录查看完整内容，已提取公开可见部分 |

## 注意事项

- 微博的账号 UID 需要在第一次访问时确认（用户名不一定等于 UID）
- 微博有防爬机制，频繁访问可能触发验证码，Agent 应采用合理频率访问
- 转发内容（repost）应标注为转发，摘要应包含原文摘要而非转发评论
