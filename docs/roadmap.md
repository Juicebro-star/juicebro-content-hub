# 演进路线（Roadmap）

本文档描述 `juicebro-content-hub` 的三阶段演进路线，从声明式中台 Skill（V1）到主动推送服务（V3）。

---

## V1：声明式中台 Skill（当前版本）

**目标**：让 Agent 能够通过本地上下文（账号矩阵、平台规则、提示模板）回答关于果汁哥内容的各类问题。

**核心能力**：
- [x] 账号矩阵完整定义（12 个平台）
- [x] 意图识别路由（6 种意图类型）
- [x] 平台适配说明（每个平台的查询策略和字段映射）
- [x] 标准化输出 Schema（NormalizedPost）
- [x] 跨平台去重策略定义
- [x] 多维汇总提示模板（按时间/平台/主题/形态）
- [x] 安全与合规声明
- [x] 导航推荐能力

**依赖**：
- Agent 自身的公开内容访问能力
- 无需额外 API 凭证

**局限**：
- 内容实时性取决于 Agent 的访问能力
- 无缓存机制，每次查询重新访问
- 部分平台（知识星球、微信私域）天然受限

---

## V2：半结构化数据接入层

**目标**：引入可选的轻量数据接入层，为 Agent 提供预处理的标准化内容缓存，提升查询效率和内容覆盖率。

**规划能力**：

### 2.1 RSS / Atom 接入
- 对支持 RSS 的平台（B 站、部分播客平台）实现 RSS 订阅拉取
- 定期缓存到本地 JSONL 文件
- Agent 优先查本地缓存，降低对实时网络访问的依赖

### 2.2 官方 API 适配
- 接入支持公开 API 的平台（B 站公开 API、小宇宙 RSS、喜马拉雅公开接口）
- 实现 `scripts/optional_fetcher.py` 的具体平台 `fetch_*` 方法
- 标准化为 NormalizedPost 格式写入本地缓存

### 2.3 主题分类器完善
- 完善 `scripts/topic_classifier.py`，基于关键词 + 简单 NLP 实现精准主题标注
- 支持多标签分类（一条内容可同时属于多个主题）

### 2.4 去重引擎实现
- 实现 `scripts/dedupe_posts.py` 中基于内容指纹的去重逻辑
- 支持合并同源内容，生成 `also_posted_on` 字段

### 2.5 增量更新机制
- 记录每个平台的最后更新时间戳
- 仅拉取增量内容，避免重复处理历史数据

**交付物**：
- `scripts/` 目录下各脚本的完整实现
- 本地缓存格式规范文档
- 增量更新调度说明（可配合 cron 或 Agent 定时任务）

**新增依赖**：
- `requests` / `httpx`（HTTP 访问）
- `feedparser`（RSS 解析）
- `jieba`（中文分词，用于主题分类）

---

## V3：主动推送与订阅服务

**目标**：从被动查询转向主动推送，实现内容监控、定时播报、跨平台聚合订阅。

**规划能力**：

### 3.1 内容更新监控
- 定时检测各平台新内容（可配置检测间隔）
- 检测到更新后主动触发 Agent 通知

### 3.2 定时日报/周报生成
- 每日自动生成"果汁哥内容日报"
- 每周自动生成"果汁哥内容周报"
- 支持输出到指定渠道（Markdown 文件、Webhook、Slack 等）

### 3.3 主题订阅
- 用户可订阅特定主题（如"比特币"、"AI"）
- 当指定主题有新内容时触发推送

### 3.4 多用户支持
- 支持多套账号矩阵配置
- 可扩展为"任意 KOL 内容聚合 Skill"（泛化版）

### 3.5 内容存档与检索
- 全量内容存档为向量数据库
- 支持语义搜索（"果汁哥有没有说过关于 AI 泡沫的观点"）

### 3.6 Skill 市场发布
- 发布到 Claude Code Skill 市场
- 发布到 OpenClaw 社区 Skill 仓库
- 提交 Hermes 兼容 Skill 认证

**新增依赖**：
- 向量数据库（Chroma / Qdrant / SQLite with vector extension）
- 嵌入模型（bge-m3 或 Claude Embeddings API）
- 消息推送适配器（Webhook / Slack / 企业微信）

## 贡献指南

欢迎社区贡献！以下是优先欢迎的贡献方向：

1. **新增平台适配说明**：若有平台我们遗漏，欢迎提 PR 添加 `adapters/` 文件
2. **完善 topic_keywords**：补充更多主题关键词映射
3. **改进提示模板**：若某类查询输出质量不佳，欢迎优化 `prompts/*.md`
4. **实现辅助脚本**：完善 `scripts/` 下的骨架脚本（遵循安全原则）
5. **添加测试用例**：为各意图类型添加覆盖用例

提交 PR 前请阅读 [docs/safety.md](./safety.md)，确保贡献符合安全原则。
