# 支持的查询意图完整清单

本文档列出 `juicebro-content-hub` Skill 支持的全部自然语言查询形式，按意图类型分组，并标注内部意图 ID 以便 Agent 路由。

---

## 一、全平台总查（intent: all_platform_query）

遍历所有可查询平台，汇总指定时间范围内的更新。

| 查询示例 | 时间范围 |
|----------|----------|
| 果汁哥今天发了什么 | today |
| 果汁哥今天更新了什么 | today |
| 果汁哥今天有什么新内容 | today |
| 果汁哥今天发布了什么 | today |
| 果汁哥最近发了什么 | last_3_days |
| 果汁哥最近三天发了什么 | last_3_days |
| 果汁哥这三天有什么更新 | last_3_days |
| 果汁哥最近一周发了什么 | last_7_days |
| 果汁哥最近七天更新了什么 | last_7_days |
| 果汁哥上周发了什么 | last_7_days |
| 果汁哥最近一个月发了什么 | last_30_days |
| 果汁哥本月有哪些更新 | this_month |
| 果汁哥最近哪个平台最活跃 | last_7_days（活跃度分析） |
| 果汁哥各平台发帖频率怎么样 | last_7_days（频率分析） |
| 果汁哥最近在哪些平台发布了内容 | last_7_days |

## 查询意图路由规则

```
1. 检测平台关键词（微博/小红书/抖音/B站/雪球/头条/公众号/喜马拉雅/小宇宙）
   → 命中则路由到 platform_single_query

2. 检测时间关键词（今天/最近/三天/一周/本周/上周/本月）
   → 未命中平台关键词则路由到 all_platform_query

3. 检测主题关键词（比特币/黄金/美股/AI/特朗普 等）
   → 路由到 topic_query，结合时间/平台过滤

4. 检测内容类型关键词（视频/文章/音频/播客/短帖）
   → 路由到 content_type_query

5. 检测汇总词（整理/日报/汇总/摘要/概览）
   → 路由到 aggregation_report

6. 检测导航词（去哪/在哪/哪个平台）
   → 路由到 navigation_recommend

7. 无法识别 → fallback_response 提示用户明确意图
```
