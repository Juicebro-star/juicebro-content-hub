# 安全与合规声明

## 定位声明

`juicebro-content-hub` 是一个围绕果汁哥公开内容进行查询、聚合、汇总、导航的 Agent Skill 规范包与扩展框架。其核心交付物是账号矩阵、意图路由规范、平台适配器文档、提示模板和标准化 Schema，**不是**一个生产级自动化爬虫系统。

---

## 本 Skill 做什么

- 提供"果汁哥"各平台公开账号的账号矩阵（平台 + 账号名 + 类别 + 可访问性状态）
- 指导 Agent 如何通过公开渠道查询和访问内容
- 提供内容规范化字段提取规范和跨平台去重策略
- 生成可读性强的内容摘要和结构化报告
- 推荐适合不同内容类型的平台

---

## 本 Skill 不做什么

### 1. 不做危险系统操作

本 Skill 不执行任何系统级命令，不读写用户系统的敏感文件，不修改操作系统配置，不调用危险 shell 命令。

### 2. 不做隐蔽命令执行

所有操作均在 Agent 可见的上下文中执行，不存在后台进程、隐藏命令或不透明的副作用。

### 3. 不默认保存私密凭证

本 Skill 不要求、不接受、不存储任何平台登录凭证（用户名/密码/Cookie/API Token）。若未来集成辅助脚本，凭证必须由用户显式提供且仅在当前会话中使用，不持久化。

### 4. 不绕过平台登录

若某平台内容需要登录才能访问（如知识星球付费内容、微信公众号部分限流内容），本 Skill 会明确告知用户该平台当前访问受限，**不会编造结果，不会尝试绕过**。

### 5. 不抓取非公开内容

本 Skill 仅聚合平台公开可见内容，不抓取私信、付费专区、需要特定权限才能访问的内容。

### 6. 不包含生产级爬虫代码

`scripts/` 目录下的辅助脚本均为**占位型骨架脚本**（skeleton stubs），提供可扩展的接口定义和类型标注，**不包含真实爬虫逻辑**，不包含绕过 robots.txt 或平台反爬机制的代码。使用者在基于这些骨架做二次扩展时，应自行审查扩展代码的合规性。

### 7. 不传播虚假信息

Agent 使用本 Skill 输出的内容时，应明确标注信息来源。本 Skill 的提示模板中明确要求：若无法获取内容，应说明原因，不允许 Agent 编造或推测内容。

---

## 数据处理原则

| 原则 | 说明 |
|------|------|
| 公开优先 | 只处理平台公开可访问的内容 |
| 最小权限 | 不要求超出内容访问需求的任何系统权限 |
| 透明边界 | 明确标注哪些平台可查、哪些平台受限 |
| 不持久化 | Skill 本身不持久化用户查询记录或内容缓存 |
| 来源标注 | 所有输出内容应标注原始平台和账号 |

---

## 用户责任

使用本 Skill 时，用户和 Agent 应当：

1. 遵守各平台的使用条款（ToS）
2. 不将本 Skill 用于大规模商业数据采集
3. 不将本 Skill 的输出用于诽谤、造谣或侵权
4. 引用内容时注明来源
5. 对基于本 Skill 骨架脚本做的二次扩展，自行承担合规责任

---

## 已知平台限制

| 平台 | 限制说明 |
|------|----------|
| 知识星球 | 付费内容，公开访问受限，Skill 仅提供平台导航，不提供内容 |
| 微信（私域）| 个人微信属于私域，不可公开查询 |
| 元宝 | 公开访问能力待确认，Skill 采用保守探测策略 |
| 抖音 | 部分内容需登录，Agent 能力取决于访问环境 |
| 快手 | 同上 |
| 喜马拉雅 | 部分付费音频不可访问 |
| 微信公众号 | 通过搜狗微信搜索访问，公开程度存在限制 |

## 报告安全问题

如发现本 Skill 存在安全隐患，请通过 GitHub Issues 报告：

```
https://github.com/Juicebro-star/juicebro-content-hub/issues
```

请使用 `security` 标签，描述问题时避免公开具体漏洞细节，优先通过私信联系仓库维护者。
